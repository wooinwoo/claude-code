# Jira Integration Rules

## 이슈 생성 규칙

### 필수 확인 사항

1. **프로젝트 키 확인** - 사용자에게 반드시 확인 (추측 금지)
2. **이슈 유형 선택** - Task, Bug, Story 중 적합한 유형
3. **템플릿 준수** - 각 유형별 템플릿 형식 사용

### Task 이슈 템플릿

```
## 작업배경
왜 이 작업이 필요한가?
- [x] {기술부채/성능개선/요청사항/기타}

{배경 설명}

## 작업내용
1. {구체적 작업}

## 완료조건
- [ ] {조건}
- [ ] 테스트 통과
- [ ] 코드 리뷰 완료

## 참고자료
- {링크}
```

### Bug 이슈 템플릿

```
## 현상
{문제 설명}

## 재현 방법
1.

## 기대 동작
{정상 동작}

## 원인 분석
{원인 또는 "분석 필요"}

## 참고자료
-
```

### Story 이슈 템플릿

```
## 사용자 스토리
As a {사용자},
I want {기능},
so that {가치}.

## 인수 조건
- [ ] Given/When/Then

## 참고자료
-
```

## MCP 도구 사용

### 이슈 생성

```typescript
mcp__jira__jira_create_issue({
  project_key: "반드시 사용자에게 확인",
  summary: "간결한 제목",
  issue_type: "Task" | "Bug" | "Story",
  description: "위 템플릿 형식 사용"
})
```

### 이슈 조회

```typescript
mcp__jira__jira_get_issue({
  issue_key: "PROJ-123"
})
```

## 금지 사항

- 프로젝트 키 추측 금지
- 템플릿 형식 무시 금지
- 불필요한 필드 추가 금지
